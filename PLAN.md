# 抓住大鹅（Catch The Goose）小游戏 — 开发方案

> **已定决策（2026-07-18）**：游戏名《抓住大鹅》/ Catch The Goose；目标平台**微信小游戏**（不再考虑喜马拉雅容器）；引擎 **Cocos Creator 3.8 + TypeScript**（不需要 Vite，Cocos 自带构建管线直出微信小游戏包）。

> 参考素材：`assets-demo/`（竞品截图 + 录屏）
> 技术参考：`game-demo/ximalaya/`（赵云与阿斗，LayaAir 2.x H5 小游戏，主要参考其平台接入与外围系统）

---

## 一、玩法设计（核心循环）

品类：**3D 物理堆叠 + 三消**（"抓大鹅 / 捞玉"类，竞品截图即此玩法）。

### 核心规则
1. 木盒中随机倾倒一堆 3D 物件（每种物件数量为 3 的倍数），物理引擎让它们自然堆叠、翻滚。
2. 玩家点击盒中任意物件 → 飞入底部 **7 格暂存槽**。
3. 槽内凑满 **3 个相同物件**自动消除（合并动画 + 音效 + 完成度增加）。
4. **失败**：7 格被填满且无法消除；或倒计时归零。
5. **胜利**：盒中物件全部消除（完成度 100%）。

### 进度与激励
- 顶部：倒计时（如 5:00）+ 关卡完成度百分比。
- 结算：按最高完成度给星 —— **50% ⭐ / 70% ⭐⭐ / 100% ⭐⭐⭐**，星级对应奖励（道具礼包）。
- 记录**最好成绩**，接入排行榜（好友榜 / 全服榜）。
- **每日免费挑战 3 次**，次数用完可看广告 / 分享 +1 次（拉留存和裂变）。

### 道具（3 个，与竞品对齐）
| 道具 | 图标 | 效果 |
|---|---|---|
| 移出 | 绿色回退箭头 | 把暂存槽中 3 个物件移出到临时栏，腾出空间 |
| 凑齐 | 磁铁 | 自动从盒中吸取 1 组（凑齐暂存槽中某物件的 3 连消） |
| 打乱 | 风扇 | 重新打乱盒中物品堆（解决遮挡/卡壳） |

获取途径：星级奖励、分享获得、（后期）广告激励。

### 难度曲线（关卡设计）
- 难度参数：**物件种类数 × 每种组数 × 时限 × 干扰项**。
- 示例：第 1 关 4 种 × 2 组 / 6 分钟 → 逐步加到 8~10 种 × 3 组 / 5 分钟。
- 后期可加变体：冰冻物件（需先消除相邻才解锁）、限定顺序、隐藏在盒底的"金鹅"彩蛋等。
- 大白鹅定位：**吉祥物 + 稀有目标**。抓到鹅有专属动画/语音（"菜！"气泡），可作为每关的趣味目标。

## 二、技术选型

### 推荐：Cocos Creator 3.8.x（TypeScript）
选它的理由：
- **原生 3D + 内置物理**（Bullet/ammo.js），刚体堆叠、点击射线拾取开箱即用 —— 本玩法的核心依赖。
- **一键发布微信小游戏**，同时支持 H5 / 抖音 / 支付宝小游戏，微信生态工具链最成熟（分包、首屏加载、开放数据域排行榜都有官方方案）。
- TS 开发、组件化、可视化编辑器，UI/弹窗/结算这类外围系统开发效率高。

### 备选对比
| 方案 | 优点 | 缺点 |
|---|---|---|
| **Cocos Creator 3.8**（推荐） | 3D+物理+微信发布一条龙，生态最全 | 需学编辑器工作流 |
| LayaAir 3.x | 与 demo 同门，也支持 3D | 3D 生态和文档弱于 Cocos |
| Three.js + cannon-es + 小游戏适配器 | 轻量、纯代码可控 | UI/适配/分包全手写，外围系统成本高 |

> 注：game-demo 是 LayaAir **2.x 2D**，不能直接复用到 3D 玩法；它的价值在 `dialog/`（弹窗体系）、`data/rank.json`（排行榜数据结构）、分享/授权/体力这套运营逻辑，做外围系统时照抄思路即可。

### 关键技术点
1. **物理堆叠**：每个物件一个凸包/复合碰撞体（不用精确 mesh collider，性能差），开局从盒子上方倾倒，物理稳定后降低 tick 频率。
2. **点击拾取**：屏幕射线检测 → 命中物件 → 关闭其物理 → Tween 飞向槽位。
3. **槽位逻辑**：纯数据层（数组 + 排序归类），与竞品一致：相同物件自动相邻排列，凑 3 即消。
4. **性能**：物件总数控制在 30~60 个；模型每个 ≤ 2k 三角面；合并材质、共享贴图图集；微信小游戏主包 ≤ 4MB，3D 资源走分包/远程加载。
5. **录屏参考**：`assets-demo/*.mp4` 可逐帧对照动效节奏（飞入槽位、消除弹跳、道具表现）。

## 三、美术设计

### 风格基调（对齐竞品截图）
- **场景**：红木首饰盒（俯视 45° 固定相机）+ 背景木柜/玻璃展示格，暖木色调。
- **物件**：翡翠质感（高光滑、半透绿）+ 白鹅（纯白、橙色喙脚、戴珠串）。写实偏 Q，靠材质出质感。
- **UI**：卡通描边风 —— 黄底圆角按钮、蓝色标题弹窗、绿色草地进度条、金色星星（截图里已有完整范式）。

### 素材清单与生产方式
| 类别 | 内容 | 数量 | 生产方式 |
|---|---|---|---|
| 3D 物件 | 白菜、扳指、手串、弥勒、貔貅、平安扣、大鹅等 | 8~10 种 | AI 生成（Tripo3D / Meshy 文生 3D）→ Blender 减面修整 → glTF；或外包建模 |
| 3D 场景 | 木盒（或木盒 3D + 背景用高清贴图） | 1 套 | 盒子建模，背景直接用 2D 图省性能 |
| UI 图集 | 按钮、弹窗框、道具图标 ×3、星星、进度条、槽位框 | ~30 张 | AI 生图（即梦/Midjourney）出风格稿 → 设计切图 |
| 动效 | 消除粒子、飞入拖尾、星星弹出 | 若干 | 引擎粒子 + Tween |
| 音频 | BGM ×1、点击/飞入/消除/胜利/失败/鹅叫 | ~8 条 | 素材库（爱给网等）+ AI 音效 |

### assets-demo 够不够？
**够做风格参考，不够直接使用**：截图是竞品实拍，不能商用直抄（版权风险），且缺 3D 模型源文件、UI 切图、音频。需按上表重新生产同风格原创素材。录屏是动效还原的最好参考。

## 四、开发计划（单人 / 约 4~6 周）

### 里程碑 M1：核心玩法可玩（1~1.5 周）
- Cocos 3.8 工程搭建，微信小游戏发布链路跑通
- 木盒 + 物理倾倒（先用几何体占位）
- 点击拾取 → 飞槽 → 三消 → 胜负判定

### 里程碑 M2：完整单局体验（1~1.5 周）
- 正式 3D 物件模型替换占位
- 倒计时、完成度、星级结算、失败/胜利弹窗
- 三个道具实现 + 消除动效/音效

### 里程碑 M3：外围系统（1 周）
- 关卡/难度配置表（JSON 驱动，参考 demo 的 `data/weapon.json` 配置思路）
- 每日次数、最好成绩本地存储
- 分享得道具、邀请裂变（参考 demo 的 ShareLpDialog 逻辑）

### 里程碑 M4：平台接入与上线（1~2 周）
- 微信登录/授权、开放数据域好友排行榜（参考 demo 的 rank 结构）
- 激励视频广告位（复活 / 得道具 / 加次数）
- 性能调优（真机帧率、包体分包）、提审上线

## 六、补充：此前未覆盖的关键项（2026-07-18 增补）

### 1. 上架资质与合规（周期最长，需最先启动）
- **微信小游戏 ≠ 随便发**：游戏类目上架需要 **软件著作权登记证书（软著）**，申请周期约 1~2 个月（加急更贵），这是整个项目的最长前置路径，应立即启动。
- **主体选择**：个人主体可发小游戏但类目/能力受限；企业主体才能完整开通流量主广告、虚拟支付。无内购的休闲游戏个人主体可行（流量主需累计 1000 UV 后开通）。
- **规避路径**：若资质周期不可接受，可先发 **H5 版**（自有域名/喜马拉雅等容器，参考 game-demo 的接入方式）或 **抖音小游戏**（资质要求类似，但审核节奏不同），玩法验证与上架流程并行。
- **诱导分享红线**：微信明令禁止"分享后才能获得奖励/复活"。竞品截图里"分享获得道具"在微信生态有违规下架风险 → 我们改为**激励视频广告**为主、分享仅作传播入口不强绑奖励。
- **命名与美术差异化**："抓大鹅"是品类通用词可参考，但整体 UI/物件不要与竞品一一对应复刻，建议起自有名字（如"翡翠盒子""鹅要发财"），美术保持同风格不同细节。

### 2. 字体 / 音频版权
- 中文字体商用授权是重灾区 → 只用免费可商用字体：**阿里巴巴普惠体 / 思源黑体 / 站酷系列**。
- 音频用 CC0 / 免费商用源（爱给网标注可商用的、freesound CC0），来源留档。

### 3. 死局与数值保护（玩法层查漏）
- 每种物件数量必须是 3 的倍数（生成器校验）。
- 槽位剩 1 格且盒中无可成对物件时的"必死"体验 → 打乱道具兜底 + 失败前给一次激励视频救场（+3 临时格或清空槽），这是此品类的核心变现点。
- 关卡随机需用**固定种子**：同一关所有人布局一致，才能做排行榜可比性与关卡回放调试。
- 反作弊：MVP 阶段成绩存本地 + 微信开放数据域（好友榜天然防伪）；全服榜等有后端再做。

### 4. 后端策略
- **MVP 零后端**：每日次数、最好成绩、道具数量全部本地存储（wx.storage）；好友排行榜用微信开放数据域（无需服务器）。
- 后续如需全服榜/活动/防作弊 → 微信云开发（免运维）起步即可，不必自建服务器。

### 5. 资源加载与包体红线
- 微信小游戏：主包 ≤ 4MB，整包（含分包）≤ 30MB。当前 7 件 glb 约 1~2MB，MVP 可全放包内。
- 素材增多后：3D 模型走分包或 CDN 远程加载。CDN 需要 **HTTPS + 已备案域名**（配到小游戏后台 request/downloadFile 合法域名）。
- 真机测试底线机型：iPhone 8 / 中端安卓（骁龙 6 系）保持 ≥ 30fps。

### 6. 埋点与调优
- 上线即接微信小游戏自带数据分析（留存/时长/分享率），关卡通过率埋点（每关开始/结束/失败原因），用于调难度曲线——此品类核心指标是**第 3~5 关的流失率**。
